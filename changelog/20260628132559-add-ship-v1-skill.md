# Add ship-v1 skill

- Added `skills/ship-v1/` — an anti-roadmap protocol to ship the smallest live version of a side project in a weekend, post it, set a signal checkpoint, then continue/pivot/drop.
- Designed as the complement to `goal-breakdown`: ship-v1 for unvalidated zero-user ideas, goal-breakdown for known-outcome work with a path or existing users. The cross-references between the pair were kept intentionally.
- Updated the README skills table with the new row.
- Why: covers the early, unvalidated side-project case where planning is over-engineering and shipping for signal beats a roadmap.
