# Add `ship-pr` skill

- New skill at `skills/ship-pr/SKILL.md` that goes from a dirty working tree to an open PR/MR in one pass: detect provider (GitHub/GitLab), detect repo conventions, derive branch + commit + PR content from the diff, push, open the review.
- Auto-detects `gh` vs `glab` from `git remote`; aborts on unsupported hosts or unauthenticated CLIs (never auto-installs).
- Enforces hard rules: no `Co-Authored-By: Claude`, no `🤖 Generated with` footers, no `--no-verify`, no `--amend`, no `--force`, no `git add -A`, no auto-push to default branch.
- Aborts cleanly on empty working tree, detached HEAD, in-progress rebase/merge, missing remote, or suspicious-looking files (`.env`, `*.pem`, `id_rsa*`, etc.).
- Filled a real gap — no existing skill covered the branch-commit-push-PR pipeline.
