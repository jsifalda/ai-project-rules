# Drop default `## Test plan` from MR/PR descriptions

- `general.md`: added rule — MR/PR description is `## Summary` only; never add a `## Test plan` / `## Testing` section unless explicitly requested.
- `ship-pr` skill: removed the `## Test plan` block from the PR body shape and all three `gh`/`glab` HEREDOC templates; left a note that Test plan is opt-in only.
- Why: auto-generated MRs were getting an unwanted Test plan section. Three sources reinforced it (Claude Code default, the skill, a CLAUDE.md line); this aligns them with the existing "clean MR description, no noise" rule.
