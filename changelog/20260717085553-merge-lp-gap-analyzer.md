# Merged landing-page-gap-analyzer into landing-page-copy

- **Removed** the `landing-page-gap-analyzer` skill. Its blueprint was a second copy of `landing-page-copy`'s, and the two had already drifted: Vision was added to the writer on 2026-07-13 and never reached the analyzer, so it was grading pages out of 39 against a 13-section standard the writer had abandoned. It also gated the money-back guarantee on `b2b_or_b2c`, a proxy it invented because it had no way to ask, where the writer uses the real `offer_guarantee` fact.
- **`landing-page-copy` now has two modes.** Generate is unchanged. Improve takes an existing page (pasted text, a file path, or a URL via `defuddle`) and rewrites it. Nothing was lost in the deletion — the analyzer was the only skill that could ingest a live page.
- **Both modes run one scored loop** over a single blueprint, replacing the writer's old 5-question self-check. It scores every section, rewrites what fails, re-scores, and stops when nothing improves.
- **Why a loop and not a report:** the analyzer's output was a document a human still had to apply. A loop that applies its own findings is strictly more useful than a standalone grader.
- **The part that took the work:** a naive score-rewrite-rescore loop is a no-op. The output template has a slot for every must-have in the blueprint, so a draft built from the template scores full marks against a rubric that only checks presence, and the loop exits having done nothing. Fixed by salvaging the analyzer's per-section anti-patterns (the writer's blueprint had none — they're the checks a template cannot satisfy by construction), deriving score bands from quoted evidence instead of choosing them, grading in a fresh context that can't see the author's intent, and making `BLOCKED` a lookup rather than a self-issued exemption.
- **Two failure modes closed as non-negotiables:** the loop otherwise pressures the model to invent metrics (Benefits Recap can't reach 3 without numbers) and, in improve mode, to reword real attributed testimonials to sharpen them.
- **Verified by running it, not by reading it.** Three blind-scoring runs plus a full improve-mode run on fixtures. A generate-mode draft scores 25/42 on round 1, well under the 40 that would prove the rubric toothless. Real attributed quotes came back byte-identical, and off-blueprint sections survived untouched.
- **The runs found real bugs, all fixed.** The rubric paid you to invent a number rather than bracket it honestly (bracketing failed the must-have, inventing passed it). A template-faithful page auto-failed the "Testimonials immediately precede Pricing" must-have, because the template's own order contradicted it. Navbar could never score 3, since "sticky on scroll" is unquotable in markdown. Placeholder scaffolding earned more points than real customers. The `capped` gate ran backwards and graded a worse page harder. The whole-page pass earned its keep — it caught a page that passed every Pricing and FAQ must-have while never saying what its free trial was.
- No new dependency. The `defuddle` edge moved from the analyzer to `landing-page-copy`.
